# サイト全面構築（エージェントチーム）

6つのエージェントチームを組織化し、DBの構築からページの作成まで一括で実行するスキル。

## 使い方
`/site-build [対象サイトの説明]`

## 実行手順

### Phase 1: 計画立案
1. 現在のプロジェクト構成を調査（技術スタック、既存DB、ページ構成）
2. EnterPlanMode で以下の6チーム構成で実装計画を作成:
   - **DBエンジニア**: スキーマ設計、マイグレーション、型定義、データアクセスヘルパー
   - **テクニカルSEO**: robots.ts, sitemap.ts, JSON-LD構造化データ, メタデータ最適化
   - **データ収集**: シードデータ作成、インポートスクリプト、バリデーション
   - **UI/UX**: コンポーネント設計、レイアウト、フィルター、ページネーション、ローディングUI
   - **ブランディング**: アイコン統一(lucide-react)、カラーシステム、信頼性シグナル
   - **競合差別化**: 独自コンテンツ企画、料金透明性、パフォーマンス最適化
3. ユーザーに計画を承認してもらう

### Phase 2: 並列実行
依存関係に基づいてフェーズを分割し、Agent ツールで並列実行:

```
Phase 1 (DB基盤) → Phase 2A (SEO) + Phase 2B (データ) → Phase 3A (UI) + Phase 3B (ブランド) → Phase 4 (差別化)
```

各フェーズで:
- Agent ツールを使って独立したタスクを並列実行
- TodoWrite でリアルタイム進捗管理
- 各フェーズ完了後に `npm run build` で検証

### Phase 3: マイグレーション & データ投入
1. SQLマイグレーションをSupabaseに適用（PowerShellでUTF-8クリップボードコピー）
2. シードスクリプトで初期データ投入
3. 動作確認

### Phase 4: デプロイ
1. フィーチャーブランチ作成
2. 全変更をコミット（詳細なコミットメッセージ）
3. PR作成 → マージ → Vercel自動デプロイ

## 重要ルール
- マイグレーションSQLのクリップボードコピーは `powershell.exe -Command "Get-Content ... -Encoding UTF8 | Set-Clipboard"` を使用（clip.exeは文字化けする）
- 各Agentには具体的なファイルパス、既存コンポーネント情報、技術スタック制約を渡す
- `npm run build` で全フェーズの検証を行う
