# SEO監査＆改善（エージェントチーム）

サイトのSEOを3つのエージェントで並列監査し、改善を実装するスキル。

## 使い方
`/seo-audit [対象ページやキーワード]`

## 実行手順

### Step 1: 3エージェント並列監査
Agent ツールで以下を同時実行:

1. **テクニカルSEOエージェント**
   - メタデータ（title, description, OG tags）の充実度
   - JSON-LD構造化データの正確性（Google Rich Results Test基準）
   - sitemap.xml の網羅性
   - robots.txt のルール確認
   - canonical URL, hreflang
   - Core Web Vitals に影響するコード（画像最適化、遅延読み込み）

2. **コンテンツSEOエージェント**
   - ページごとのテキスト量（薄いコンテンツがないか）
   - 見出し構造（H1→H2→H3の階層）
   - 内部リンク構造（孤立ページがないか）
   - FAQ構造化データの活用
   - ロングテールキーワードの網羅性

3. **競合差別化エージェント**
   - 競合サイトにない独自コンテンツの提案
   - ユーザー目線で足りない情報の特定
   - E-E-A-T（経験・専門性・権威性・信頼性）の向上ポイント

### Step 2: 優先度付き改善リスト作成
3エージェントの結果を統合し、TodoWriteで優先度順に整理

### Step 3: 改善実装
Agent ツールで並列に修正を実装

### Step 4: ビルド検証
`npm run build` で動作確認
